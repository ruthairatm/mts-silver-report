Acceptance Criteria

- **Fit In 1:1 Main Image**: portrait (3:4) main image renders as 1:1 Fit-In (not cropped), remaining area filled with white background. - **Crop Center Thumbnail**: thumbnail carousel images render as Crop Center, filling the square frame with no white border. - **First Image Auto Crop**: the first image (index 0) is always forced into a 1:1 frame to leave room for campaign tag overlays. - **Max 20 Files Logic**: if API returns more than 20 images/videos combined, only the first 20 are rendered; files 21+ never appear. - **Separate Thumbnail Layout**: the thumbnail carousel is laid out as a visually separate block from the main gallery image. - **No Autoplay**: any video in the gallery is paused by default; playback starts only when the user taps the Play button. - **Video Blurred Background**: a 9:16 video is Fit-In centered with a blurred, scaled copy of itself as the background fill. - **1-Image Edge Case**: if the product has exactly 1 image (no video), the thumbnail carousel and dot indicator/count are hidden and swipe is disabled. - **Fullscreen Pinch-to-Zoom**: tapping the main image opens a fullscreen lightbox supporting pinch-to-zoom and swipe. - **Variant Thumbnail Sync**: selecting a variant (e.g. color) auto-slides the main gallery to the matching image index. - **Modern Format Render**: system must fully render `.webp` main images and `.webm` videos (plus `.jpg/.jpeg`, `.png`, `.mp4` per spec). - **(Negative) API Error Fallback**: if the image API fails (404/500), show an empty 1:1 placeholder instead of crashing/breaking the layout.
